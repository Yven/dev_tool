## 一、技术选型

### 1.1 桌面框架对比

| 指标 | Tauri v2 | Electron 33 | C# WPF (.NET 8) | Wails (Go) |
|------|----------|-------------|------------------|------------|
| 冷启动(HDD) | ~300-500ms | ~1500-3000ms | ~400-800ms | ~500-800ms |
| 空载内存 | ~15-25MB | ~50-80MB | ~20-40MB | ~25-40MB |
| 安装包大小 | ~2-4MB | ~50-70MB | ~15-30MB | ~8-12MB |
| 全局热键 | ✅ 官方插件 | ✅ 内置 | ✅ Win32 API | ✅ 原生 |
| 多屏支持 | ✅ monitor API | ✅ screen API | ✅ WPF原生 | ✅ 原生 |
| 插件体系 | ✅ 官方架构 | ⚠️ 需自建 | ⚠️ MEF/自建 | ⚠️ 需自建 |
| 贴边隐藏 | ✅ 窗口API实现 | ✅ 可实现 | ✅ 可实现 | ✅ 可实现 |
| 静默安装 | ✅ NSIS /S | ✅ 可配置 | ✅ MSI /quiet | ⚠️ 需自建 |
| Win10 1903+ | ✅ WebView2 | ✅ Chromium | ✅ .NET运行时 | ✅ WebView2 |
| 生态活跃度 | 高(快速增长) | 高(成熟) | 高(成熟) | 中 |

**结论：Tauri v2** — 唯一同时满足三项硬性指标（启动≤800ms / 内存≤35MB / 安装包≤8MB）的方案。

### 1.2 前端框架选型

| 框架 | 运行时(gzip) | 性能 | TypeScript | 适合插件UI |
|------|-------------|------|-----------|-----------|
| SolidJS | ~7KB | 极优(细粒度响应) | ✅ 一等支持 | ✅ |
| Svelte 5 | ~2KB | 极优(编译期优化) | ✅ 良好 | ⚠️ 编译限制 |
| Vue 3 | ~33KB | 优 | ✅ 一等支持 | ✅ |
| React | ~42KB | 良 | ✅ 一等支持 | ✅ |
| Vanilla | 0KB | - | ✅ | ❌ |

**结论：SolidJS** — 运行时极小(7KB)，细粒度响应式天然适合实时转换场景，TypeScript 支持完善，JSX 语法降低学习成本，组件模型适合插件 UI 注入。

### 1.3 CSS 方案

**结论：UnoCSS** — 原子化 CSS 引擎，按需生成，零运行时，产物极小(~2-5KB)，支持自定义 preset 实现原生 Windows 视觉风格。

### 1.4 构建工具

**结论：Vite** — Tauri 官方推荐，HMR 极快，SolidJS 官方支持，插件生态丰富。

### 1.5 语言

| 层 | 语言 | 理由 |
|----|------|------|
| 后端(Rust侧) | Rust | Tauri 强制，性能+安全 |
| 前端 | TypeScript | 类型安全，SolidJS/JSX 原生支持 |
| 插件SDK | TypeScript | 降低插件开发门槛 |

### 2.2 进程模型

```
Main Process (Rust)
├── 窗口管理：创建/销毁/贴边隐藏/多屏计算
├── 全局热键：注册/响应/自定义
├── 插件加载器：扫描/注册/沙箱/热插拔
├── 单实例守卫：tauri-plugin-single-instance
├── 剪贴板：tauri-plugin-clipboard-manager
├── 文件系统：tauri-plugin-fs (插件目录读写)
└── 自动更新：tauri-plugin-updater

WebView Process (Chromium/WebView2)
├── SolidJS 应用壳
├── 4个内置模块组件
├── 插件 UI 渲染容器
└── Plugin SDK 运行时
```

### 2.3 数据流

```
用户输入 → SolidJS组件 → invoke("cmd_xxx", args) → Rust Command → 计算结果 → IPC返回 → 组件更新 → 渲染输出
```

## 四、核心模块设计

### 4.1 贴边隐藏 (Edge Hide)

**原理**：窗口贴边时缩小为 3px 触发条，鼠标进入触发条区域后展开完整窗口。

```
Rust 侧 (src-tauri/src/window/edge_hide.rs):
├── EdgeSide 枚举: Top / Left / Right / None
├── EdgeHideManager 结构体
│   ├── detect_edge() -> EdgeSide     # 检测窗口是否贴边
│   ├── collapse_to_trigger()         # 缩为 3px 触发条
│   ├── expand_full()                 # 展开完整窗口
│   ├── on_mouse_enter_trigger()      # 鼠标移入(120ms防抖)
│   └── on_mouse_leave_window()       # 鼠标移出(400ms延迟)
├── 多屏适配
│   ├── get_available_monitors()       # 获取所有显示器
│   ├── clamp_to_visible_area()       # 防止窗口超出可视区域
│   └── get_nearest_edge()            # 计算最近屏幕边缘
└── 与 Tauri 窗口 API 交互
    ├── window.set_position()
    ├── window.set_size()
    └── window.on_window_event()
```

**关键实现**：
- 监听 `WindowEvent::Moved` 判断是否贴边
- 贴边后设置窗口尺寸为 `(3px, screenHeight)` 或 `(screenWidth, 3px)`
- 使用 `set_decorations(false)` 去除标题栏
- 鼠标事件通过 WebView 的 `mouseenter`/`mouseleave` 触发 IPC 通知 Rust 侧
- 使用 `tauri_plugin_positioner` 辅助窗口定位

### 4.2 全局快捷键

```rust
// src-tauri/src/hotkey/manager.rs
// 使用 tauri-plugin-global-shortcut
use tauri_plugin_global_shortcut::{Code, GlobalShortcutExt, Modifiers, Shortcut, ShortcutState};

pub struct HotkeyManager {
    current_key: String,  // 默认 "Ctrl+Shift+Space"
}

impl HotkeyManager {
    pub fn register(&self, key: &str) -> Result<()>    // 注册快捷键
    pub fn unregister(&self, key: &str) -> Result<()>  // 注销
    pub fn update_key(&mut self, old: &str, new: &str)  // 更新(先注销再注册)
    pub fn on_triggered() -> toggle窗口显示/隐藏
}
```

**Rust 侧注册示例**（基于 Tauri v2 官方插件 API）：
```rust
let shortcut = Shortcut::new(Some(Modifiers::CONTROL | Modifiers::SHIFT), Code::Space);
app.global_shortcut().register(shortcut)?;
// handler 中判断 ShortcutState::Pressed 时 toggle 窗口
```

**前端侧注册示例**：
```typescript
import { register } from '@tauri-apps/plugin-global-shortcut';
await register('CommandOrControl+Shift+Space', (event) => {
  if (event.state === 'Pressed') {
    // toggle window visibility
  }
});
```

### 4.3 插件加载器

```
Rust 侧 (src-tauri/src/plugin_loader/):
├── scanner.rs
│   ├── scan_plugins_dir()            # 扫描 plugins/ 目录
│   ├── validate_plugin(file)         # 验证: ESModule, ≤200KB
│   └── watch_changes()              # fs watcher 实现热插拔
├── registry.rs
│   ├── PluginMeta { id, name, version, entry, icon }
│   ├── register(meta)               # 注册到内存表
│   ├── unregister(id)               # 从内存表移除
│   └── list_all() -> Vec<PluginMeta>
└── sandbox.rs
    ├── 创建隔离的 JS 沙箱(iframe sandbox)
    ├── 暴露 PluginSDK: clipboard, invoke, storage
    └── 禁止: Node API, 文件系统直接访问, 网络请求(除非声明)
```

**插件规范**：
```typescript
// 插件入口文件示例 (my-plugin.js)
export default {
  id: 'my-plugin',
  name: 'My Plugin',
  version: '1.0.0',
  icon: '📝',
  render: (container: HTMLElement, sdk: PluginSDK) => {
    // 使用 SDK 提供的能力渲染 UI
  },
  destroy: () => {
    // 清理资源
  }
}
```

**PluginSDK 接口**：
```typescript
interface PluginSDK {
  clipboard: {
    readText(): Promise<string>
    writeText(text: string): Promise<void>
  }
  storage: {
    get(key: string): Promise<string | null>
    set(key: string, value: string): Promise<void>
  }
  invoke(cmd: string, args?: Record<string, unknown>): Promise<unknown>
}
```

### 4.4 四个内置模块

**计算逻辑放在 Rust 侧（通过 Tauri Commands），前端只做 UI 渲染和事件绑定。**

| 模块 | Rust Command | 前端组件 | 关键逻辑 |
|------|-------------|---------|---------|
| 时间戳转换 | `cmd_timestamp_convert` | `TimestampPanel.tsx` | 秒/毫秒/微秒互转，时区偏移计算，格式化输出 |
| UTF-8中文转换 | `cmd_encoding_convert` | `EncodingPanel.tsx` | 自动检测\uXXXX/&#x/%E，双向编解码 |
| URL编解码 | `cmd_url_encode` / `cmd_url_decode` | `UrlPanel.tsx` | RFC 3986实现，保留字符白名单，批量多行 |
| JSON格式化 | `cmd_json_format` | `JsonPanel.tsx` | serde_json解析，错误行号定位，压缩/排序/Unicode转义 |

**为什么计算逻辑放 Rust**：
1. 性能：JSON 大文件格式化、批量 URL 编解码在 Rust 侧更快
2. 安全：插件沙箱无法直接调用 Rust Command（需通过 SDK 白名单）
3. 测试：Rust 侧可独立做单元测试，不依赖 UI

**Rust Command 示例**：
```rust

fn cmd_timestamp_convert(input: &str, unit: &str, timezone: &str) -> Result<TimestampResult, String> {
    // 解析输入 → 转换 → 格式化 → 返回
}

#[tauri::command]
fn cmd_json_format(input: &str, indent: u32, sort_keys: bool, escape_unicode: bool) -> Result<JsonResult, String> {
    // serde_json::from_str → 格式化/排序/转义 → 返回
    // 解析失败时返回错误行号 + 原因
}
```

### 6.3 NSIS 安装配置

```json
// tauri.conf.json > bundle.windows.nsis
{
  "installMode": "currentUser",
  "languages": ["SimpChinese", "English"],
  "displayLanguageSelector": true
}
```

- NSIS 原生支持 `/S` 静默安装参数
- 预估安装包大小：Rust 二进制(~1.5MB) + WebView2引导(~0.5MB) + 前端资源(~0.5MB) + NSIS壳(~0.3MB) ≈ **~3MB**（远低于 8MB 限制）
- 同时输出 Portable ZIP 绿色版

### 6.4 WebView2 处理

- Win10 1903+ 已内置 WebView2 Runtime
- NSIS 安装脚本中添加 WebView2 检测，缺失时引导下载或内嵌 bootstrapper
- 可通过 NSIS Hook `!macro NSIS_HOOK_POSTINSTALL` 检测并安装 WebView2

## 十一、验证方式

1. **启动性能**：`Measure-Command { Start-Process dev_tool.exe -Wait }` 测量冷启动；热启动通过全局热键触发计时
2. **内存占用**：`Get-Process dev_tool | Select-Object WorkingSet64` 空载状态下测量
3. **安装包大小**：`ls -la` 检查 NSIS 输出 ≤8MB
4. **功能验收**：Playwright E2E 测试覆盖 4 模块 + 贴边隐藏 + 热键
5. **插件热插拔**：放入/删除 .js 文件，验证自动加载/卸载
6. **多屏适配**：双屏环境下拖拽窗口至各边缘，验证贴边行为
